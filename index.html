Quick Start Guide
このアプリの使い方
ビール製造の全工程（仕込み→缶充填）に渡る55件の検査基準を管理するHACCPナレッジベースです。
Notionと双方向で連携しており、アプリ上での変更が自動でDBに反映されます。
基本的な使い方 — 4ステップ
01
用語解説で全体像を把握
初めての方は「📖 用語解説」タブを開き、CCP・OP・Q の違いとHACCP7原則を確認してください。 右のサイドバーからも各用語にジャンプできます。
💡 専門用語（CIP・TPO・IBUなど）は用語集で解説しています
02
基準一覧を確認・検索
「📋 基準一覧」タブでNotionDBの全項目を読み込みます。サイドバーの工程フィルターや上部のCCP/OP/Q フィルターで絞り込めます。
🔍 検索ボックスで項目名・工程をリアルタイム検索できます
03
項目をクリックして詳細ページへ
各行をクリックすると「📄 詳細ページ」が開きます。工程フロー・NG時処置・チェックリスト・メモ欄が使えます。 相互参照から同工程や同分類の他項目に素早く移動できます。
✏️ 「編集」ボタンで基準値の変更が可能。変更履歴は自動でNotionに記録
04
編集・追加でNotionに保存
一覧の「＋ 追加」ボタンで新規項目、詳細の「✏️ 編集」で既存項目を変更できます。 保存すると🔬 基準マスタDBと📋 変更履歴DBの両方に反映されます。
💾 詳細ページの「メモ」もNotionのフィールドに保存されます

詳細ページの各セクション
⚠️
分類の理由説明
CCP・OP・Qそれぞれなぜその分類なのかを説明します。CCPなら「後工程での回収不可」、OPなら「補正余地あり」など。監査時の説明根拠になります。
📊
判定基準値カード
緑 = 合格（CL）・黄 = 警告値・赤 = NGライン の3段階で視覚化。測定方法・頻度・改定バージョンも確認できます。
🔄
工程フロー図
仕込みから出荷前まで全7工程を横断したフロー図で、現在の項目の位置をハイライト表示します。各工程に同分類の項目が何件あるかも一目で確認できます。
✅
NG時処置チェックリスト
逸脱発生時の対応手順がチェックリスト形式で表示されます。CCP/OP/Qでデフォルト手順が異なります。チェックを入れると打ち消し線で完了を記録できます。
📝
作業メモ・手順詳細エディタ
「詳細手順」タブには測定の具体的ステップ、「備考」タブには注意事項や過去の逸脱事例などを自由記入できます。「Notionに保存」でDBのフィールドに反映されます。
🔗
相互参照
同じ工程の他の検査項目と、他工程の同分類（CCP/OP/Q）項目がカードで並びます。クリックで即座に詳細ページへ遷移できます。工程をまたいだ管理点の整合確認に役立ちます。
よくある質問
Q Notionとの接続が確立されるまでどれくらいかかりますか？ ▾
「📋 基準一覧」タブを初めて開いたときに読み込みが始まります。Claude APIを経由してNotionのMCPサーバーに接続するため、初回は20〜40秒程度かかります。右上のインジケーターが緑に変わると接続完了です。
Q 基準値を変更したら変更履歴は自動で残りますか？ ▾
はい。「Notionに保存」ボタンを押すと、検査基準マスタDBの更新と同時に、変更履歴DBへ「変更前→変更後」「変更者」「変更理由」「日時」「バージョン」が自動で書き込まれます。バージョン番号もv1.0→v1.1のように自動インクリメントされます。
Q 詳細ページに入力したメモはどこに保存されますか？ ▾
「詳細手順」は検査基準マスタDBの「測定方法」フィールドに、「備考」は「備考」フィールドにそれぞれ保存されます。Notionのページを直接開いても同じ内容が確認できます。「Notionで開く ↗」ボタンからジャンプできます。
Q 新しい検査項目を追加するにはどうすればよいですか？ ▾
「📋 基準一覧」の右上にある「＋ 追加」ボタンをクリックします。入力フォームが開くので、検査項目名・工程・分類（必須）と各基準値を入力し「Notionに保存」を押してください。追加後は自動で一覧が更新されます。
Q GASとの違いは何ですか？ ▾
このアプリはブラウザから直接Notionに接続する「閲覧・編集ツール」です。GAS（Google Apps Script）は主にGoogleスプレッドシートからNotionへ一括データを送信するための「バッチ投入ツール」です。定期的な大量更新はGAS、日常的な参照・個別編集はこのアプリという使い分けを推奨します。
HACCP とは
Hazard Analysis and Critical Control Points
食品の安全性を確保するための国際標準的な管理システム。「どこで何が危険になるか」を事前に分析し、その重要な制御点（CCP）を継続的に監視・記録する手法。 製造工程全体を「危害が発生するリスク」「後工程で除去できるか」の2軸で評価し、科学的根拠に基づく基準を設定する。

日本では2021年より食品衛生法に基づくHACCP義務化が完全施行されており、規模を問わず全ての食品事業者に適用される。
分類の違い — CCP / OP / Q
CCP
重要管理点
Critical Control Point
食品安全上の危害を防止・除去・許容範囲内に低減するために必須の工程。
ここが崩れると後工程での回復が不可能。監査で最優先に確認される。
本DBの例
煮沸温度（殺菌の唯一の機会）
発酵タンクATP（CIP後の清潔確認）
ヘッドスペースDO・シーム強度（缶充填）
アレルゲン表示確認（食品表示法）
⚠️ CLを逸脱したら即座に是正措置が必要。記録の欠落も違反とみなされる。
OP
一般管理点
Operational Pre-Requisite Program
食品安全に関連するが、CCPほど絶対的ではない管理点。逸脱しても後工程で補正や回収の余地がある場合が多い。
PRPと組み合わせてGMP（適正製造規範）を構成する。
本DBの例
仕込み水温度（レシピ±2℃ 後で調整可）
冷却所要時間（60分以内）
官能評価・異臭チェック
貯酒温度（0–4℃）
ℹ️ 逸脱時は記録＋上長報告。製品は後工程での確認後に判断。
Q
品質管理点
Quality Control Point
食品安全への直接的影響は低いが、製品品質・風味・外観に影響する管理点。HACCPの7原則には含まれないが、ブランド品質の維持に不可欠。
本DBの例
麦汁OG・FG（目標比重±2P）
炭酸ガス圧（CO₂ Vol）
濾過後濁度（EBC値）
発酵後ABV（推定アルコール度数）
✦ 逸脱時は品質低下として記録。安全問題ではないが官能評価強化を行う。
判定基準の3段階
合格 CL
Critical Limit（臨界限界値）
この範囲内であれば製品は安全・品質適合とみなされる数値範囲。科学的根拠に基づき設定。
警告値
Warning Level（運用限界値）
CLに近づいているが未逸脱の状態。上長報告・操作調整が必要。記録に残し次の測定で確認する。
NG ライン
Critical Limit 逸脱
CLを超えた状態。CCPの場合は即座に是正措置を発動し、当該ロットを保留・調査する。
HACCP 7原則
1
危害要因分析（HA） — 生物的・化学的・物理的危害を工程ごとに特定し、重大性と発生可能性を評価する
2
CCP の決定 — 決定木（ディシジョンツリー）を使い、どの工程がCCPに該当するかを判断する
3
管理基準（CL）の設定 — 各CCPに対して科学的根拠のある合格限界値（臨界限界値）を設ける
4
モニタリング手順の確立 — 誰が・何を・どのくらいの頻度で・どうやって測定するかを明文化する
5
是正措置の確立 — CL逸脱時に取るべき具体的な行動手順（保留・廃棄・原因調査）を事前に定める
6
検証手順の確立 — HACCPシステムが有効に機能していることを定期的に確認・見直しする
7
文書化・記録保管 — 全ての管理記録を適切に保管し、監査や問題発生時のトレーサビリティを確保する
専門用語集
CIP
定置洗浄
Clean In Place。タンクや配管を分解せずに内部を循環洗浄・殺菌する方式。ATPで洗浄効果を数値確認する。
TPO
缶内総酸素量
Total Package Oxygen。缶充填時にヘッドスペースに残留する酸素量（ppb）。高いと酸化劣化・賞味期限短縮の原因。
OG / FG
初期比重 / 終末比重
Original / Final Gravity。麦汁の糖濃度を示す比重。OGとFGの差からABV（アルコール度数）が算出される。
IBU
国際苦味単位
International Bitterness Units。ホップ由来のイソα酸濃度を示す。ホップ添加量・タイミング・煮沸時間に依存する。
DO / RLU
溶存酸素 / ATP発光量
Dissolved Oxygen（ppm）は酸化の主因。RLU（Relative Light Unit）はATP検査での清潔度指標。≤200RLUが合格。
EBC
欧州醸造規格（濁度・色度）
European Brewery Convention。ビールの濁度・色度の国際標準単位。クリア系ビールは≤0.5 EBC。
ABV
アルコール度数
Alcohol By Volume（%）。酒税区分・ラベル表示に直結。OG-FG差から推定式で計算し、実測との照合も行う。
Lot / トレーサビリティ
ロット管理・追跡可能性
原材料の入荷Lotから製品出荷Lotまでを紐付けて追跡できる仕組み。問題発生時の迅速なリコール対応に必要。
是正措置 / 有効性確認
CAPA
Corrective And Preventive Action。問題発生時に原因を特定・修正し、再発防止策が機能しているかを後日確認するプロセス。
